Why TunnelEdge

Built for developers,
not DevOps teams

Everything you need to share your work with the world — without touching a single cloud console.

QUIC Transport

Built on HTTP/3's transport layer. Multiplexed streams, 0-RTT reconnection, and zero head-of-line blocking across all your tunnels.

Secure by Default

Mutual TLS with auto-provisioned wildcard certificates. Every agent authenticates with bcrypt-hashed tokens. Nothing in plaintext, ever.

Multiple Tunnels Per Agent

One agent, unlimited tunnels. Route api.<agent>.dev to port 3000 and grpc.<agent>.dev to 50051 simultaneously.

Live Dashboard

Real-time session monitoring via Server-Sent Events. See active connections, bandwidth telemetry, and connection status without refreshing.

Self-Hosted First

Deploy on your own infra with Docker Compose in under 5 minutes. Your data, your keys, your infrastructure. No SaaS lock-in.

Interactive TUI Agent

A full terminal UI for your agent — QUIC stream visualizer, bandwidth sparkline, structured log viewer, and live connection state in your terminal.

Simple setup

Up and running in 3 steps

1

Deploy the gateway

Spin up registry + gateway + dashboard with a single Docker Compose command on any server.

shell 
docker compose up -d
2

Create an agent

Register your machine in the dashboard, grab the token, and drop it in your config file.

agent.yaml 
token: te_xxxxxxxxxxxxxxxx
tunnels:
  - label: web
    local_addr: localhost:3000
3

Go live

Run the agent. Your service is instantly reachable at a public URL — no DNS changes, no firewall rules.

result 
✓ https://web-myserver.yourdomain.dev
